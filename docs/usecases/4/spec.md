# UC-004: 광고주 정보 등록

## 1. 유스케이스 정보

| 항목 | 내용 |
|------|------|
| **유스케이스 ID** | UC-004 |
| **유스케이스 이름** | 광고주 정보 등록 |
| **액터** | 로그인된 사용자 (역할 미등록) |
| **우선순위** | 높음 |
| **관련 기능** | 사용자 정보 관리 |

## 2. 사전조건 (Preconditions)

- 사용자가 로그인 상태
- User.role이 NULL (역할 미등록 상태)
- 광고주 정보가 등록되지 않음

## 3. 사후조건 (Postconditions)

### 성공 시
- User 테이블 업데이트 (이름, 생년월일, 휴대폰번호)
- Advertiser 테이블에 레코드 생성
- User.role이 'advertiser'로 업데이트
- 세션에 역할 정보 저장
- 광고주 대시보드로 리다이렉트

### 실패 시
- 에러 메시지 표시
- 광고주 정보 등록 페이지 유지

## 4. 주요 플로우 (Main Flow)

1. 사용자가 광고주 정보 등록 페이지(`/advertiser/register`)에 접속
2. 다음 정보 입력:
   - **공통**: 이름, 생년월일 (YYYY-MM-DD), 휴대폰번호 (010-XXXX-XXXX)
   - **광고주 전용**: 업체명, 주소 (우편번호 + 상세주소), 업장 전화번호, 사업자등록번호 (XXX-XX-XXXXX), 대표자명
3. "등록" 버튼 클릭
4. **시스템**: 로그인 여부 검증
5. **시스템**: 중복 등록 검증
   - Advertiser 테이블에서 user_id로 레코드 조회
   - 이미 등록된 경우: "이미 광고주 정보가 등록되어 있습니다" + 홈으로 리다이렉트
6. **시스템**: 입력 검증
   - 이름: 2자 이상, 한글/영문만
   - 생년월일: YYYY-MM-DD 형식, 만 19세 이상
   - 휴대폰번호: 010-XXXX-XXXX 형식, 중복 여부 확인
   - 업체명: 2자 이상
   - 주소: 우편번호, 기본주소, 상세주소 입력 확인
   - 업장 전화번호: XXX-XXXX-XXXX 형식
   - 사업자등록번호: XXX-XX-XXXXX 형식, 중복 여부 확인
   - 대표자명: 2자 이상, 한글/영문만
7. **시스템**: 트랜잭션 시작
8. **시스템**: User 테이블 업데이트
   - name, birth_date, phone_number 업데이트
   - role을 'advertiser'로 업데이트
9. **시스템**: Advertiser 테이블에 레코드 생성
   - user_id, business_name, address, business_phone, business_number, representative_name 저장
10. **시스템**: 트랜잭션 커밋
11. **시스템**: 세션에 역할 정보 저장 (role: 'advertiser')
12. **시스템**: 광고주 대시보드(`/advertiser/dashboard`)로 리다이렉트
13. **시스템**: "광고주 정보가 등록되었습니다" 메시지 표시

## 7. 비즈니스 규칙 (Business Rules)

### BR-001: 사업자등록번호 중복 불가
- 동일한 사업자등록번호는 한 번만 등록 가능
- Advertiser.business_number UNIQUE 제약조건

### BR-002: 만 19세 이상만 광고주 가능
- 생년월일 기준 만 19세 미만은 광고주 등록 불가
- 애플리케이션 레벨에서 검증

### BR-003: User와 Advertiser 1:1 관계
- 한 User는 하나의 Advertiser 레코드만 가질 수 있음
- Advertiser.user_id UNIQUE 제약조건

### BR-004: 트랜잭션 원자성
- User 업데이트와 Advertiser 생성은 트랜잭션으로 묶어 원자성 보장
- 하나라도 실패 시 전체 롤백

- 업장 전화번호: 텍스트 입력 (XXX-XXXX-XXXX)
- 사업자등록번호: 텍스트 입력 (XXX-XX-XXXXX)
- 대표자명: 텍스트 입력

### 버튼
- "등록" 버튼 (Primary)

### 메시지
- 성공: "광고주 정보가 등록되었습니다"

## 10. 시퀀스 다이어그램

```
사용자 -> 광고주 정보 등록 페이지: 접속
광고주 정보 등록 페이지 -> 사용자: 입력 폼 표시
사용자 -> 광고주 정보 등록 페이지: 정보 입력 및 제출
광고주 정보 등록 페이지 -> AuthService: 로그인 검증
AuthService --> 광고주 정보 등록 페이지: 검증 성공
광고주 정보 등록 페이지 -> DB (Advertiser): 중복 등록 검증
DB (Advertiser) --> 광고주 정보 등록 페이지: 중복 없음
광고주 정보 등록 페이지 -> WTForms: 입력 검증
WTForms --> 광고주 정보 등록 페이지: 검증 성공
광고주 정보 등록 페이지 -> DB: 트랜잭션 시작
광고주 정보 등록 페이지 -> DB (User): User 업데이트
DB (User) --> 광고주 정보 등록 페이지: 업데이트 완료
광고주 정보 등록 페이지 -> DB (Advertiser): Advertiser 생성
DB (Advertiser) --> 광고주 정보 등록 페이지: 생성 완료
광고주 정보 등록 페이지 -> DB: 트랜잭션 커밋
DB --> 광고주 정보 등록 페이지: 커밋 완료
광고주 정보 등록 페이지 -> Flask Session: 역할 저장
Flask Session --> 광고주 정보 등록 페이지: 저장 완료
광고주 정보 등록 페이지 -> 사용자: 광고주 대시보드로 리다이렉트
```

## 11. 테스트 시나리오

### 11.1 정상 등록
- **Given**: 로그인된 사용자 (역할 미등록)
- **When**: 모든 필드를 올바르게 입력하고 제출
- **Then**: User/Advertiser 테이블 업데이트, 광고주 대시보드로 이동

### 11.2 사업자등록번호 중복
- **Given**: 이미 등록된 사업자등록번호 존재
- **When**: 동일한 사업자등록번호로 등록 시도
- **Then**: "이미 등록된 사업자등록번호입니다" 에러 메시지 표시

### 11.3 만 19세 미만
- **Given**: 생년월일이 만 19세 미만
- **When**: 광고주 정보 등록 시도
- **Then**: "만 19세 이상만 광고주로 등록할 수 있습니다" 에러 메시지 표시

### 11.4 중복 등록 시도
- **Given**: 이미 광고주 정보가 등록된 사용자
- **When**: 광고주 정보 등록 페이지 접근
- **Then**: "이미 광고주 정보가 등록되어 있습니다" 메시지, 광고주 대시보드로 리다이렉트
